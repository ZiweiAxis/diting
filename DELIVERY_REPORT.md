🚀 核心功能

【HTTP 代理层】
  ✅ 反向代理 - 透明拦截 Agent 请求
  ✅ 请求截获 - 完整捕获 Method/Path/Body
  ✅ 响应处理 - 支持 403/200/其他状态码

【风险评估】
  ✅ 规则引擎 - 基于方法/路径/关键词的硬规则
  ✅ 三级分类 - 低/中/高风险自动识别
  ✅ 灵活配置 - YAML 配置文件

【AI 意图分析】
  ✅ LLM 集成 - 支持 Ollama 本地模型
  ✅ 意图识别 - 分析操作意图和影响
  ✅ 智能降级 - LLM 不可用时自动降级

【人工审批】
  ✅ 命令行审批 - y/n 快速确认
  ✅ 超时机制 - 自动拒绝 (计划中)
  ✅ 审批记录 - 记录审批人和时间

【eBPF 内核监控】
  ✅ execve 监控 - 命令执行追踪
  ✅ unlinkat 监控 - 文件删除追踪
  ✅ tcp_connect 监控 - 网络连接追踪
  ✅ write 监控 - 文件写入追踪
  ✅ perf event - 内核态到用户态传输

【审计溯源】
  ✅ JSONL 日志 - 结构化事件记录
  ✅ 全链路记录 - Who/What/When/Why
  ✅ 证据链存储 - 完整决策过程

【DNS 劫持架构】⭐
  ✅ DNS 劫持器 - 将 Agent 流量劫持到 WAF 网关
  ✅ WAF 网关 - 深度检测和治理
  ✅ 透明代理 - Agent 无感知，无法绕过
  ✅ 全协议支持 - HTTP/HTTPS/gRPC/TCP

🎯 技术亮点

1. eBPF 内核级监控
   - 传统监控: strace/ltrace (1000x 慢)
   - eBPF: 零拷贝，内核态执行
   - 性能: < 1% 开销，> 10k events/s

2. AI 驱动决策
   - LLM 意图分析 (50 字智能判断)
   - 本地部署，保护隐私
   - 自动降级到规则引擎

3. 零信任架构
   - 默认拒绝，显式批准
   - 持续验证，永不信任
   - 最小权限原则

4. 全链路可追溯
   - 每个操作都有完整审计
   - Who/What/When/Why
   - 满足等保 2.0 / ISO 27001

🚀 快速启动

【方式 1: Python 版本 (推荐快速测试)】

Windows:
  cd E:\workspace\sentinel-ai
  pip install requests
  python sentinel.py

Linux/Mac:
  cd /mnt/e/workspace/sentinel-ai
  pip3 install requests
  python3 sentinel.py

测试:
  curl http://localhost:8080/get           # 安全请求 (自动放行)
  curl -X DELETE http://localhost:8080/delete  # 危险请求 (需要审批)


【方式 2: Docker 部署】

Windows:
  cd E:\workspace\sentinel-ai
  docker-deploy.bat

Linux/Mac:
  cd /mnt/e/workspace/sentinel-ai
  chmod +x docker-deploy.sh
  ./docker-deploy.sh

选择选项 1 启动 Python 版本


【方式 3: eBPF 内核监控 (Linux + root)】

启动 WSL:
  wsl -d Ubuntu-24.04

安装依赖:
  sudo apt update
  sudo apt install -y bpfcc-tools linux-headers-$(uname -r)
  pip3 install bcc

运行:
  cd /mnt/e/workspace/sentinel-ai
  sudo python3 sentinel_ebpf.py
